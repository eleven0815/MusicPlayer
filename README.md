# 音栖 · Android 本地音乐播放器

“音栖”是一个仅播放设备本地音频的 Android 音乐播放器原型，视觉语言参考 QQ 音乐的轻盈层级与绿色品牌感，但所有界面、图标和媒体资产均为原创。

## Figma 原型

- 设计文件：[音栖 Android 本地音乐播放器](https://www.figma.com/design/om7vONwiMGnxmkFO73mCPI)
- 基线尺寸：`412 × 915dp`，竖屏
- 360dp 适配：页面边距缩至 16dp、双列卡片缩至 156dp、播放器封面最小 280dp
- 原型起点：首次启动与权限、完整本地曲库、沉浸播放器

## Android 实现

- 本地扫描、可见的重新扫描结果、权限拒绝与系统文件夹选择回退（SAF）
- 歌曲、专辑、歌手、文件夹、搜索、喜欢与自建歌单；专辑和歌手支持网格/列表切换
- 迷你播放器、深色沉浸播放器、封面/歌词左右滑动、队列与上下文操作菜单
- 可点击的封面/歌词标签、逐行 LRC 同步、高亮、点击歌词跳转与纯文本降级
- Room 持久化曲库和补全状态；扫描后可在 Wi‑Fi 下后台补齐缺失歌词与封面
- 本地优先封面链路、原创缺图封面、封面取色动态背景与通知栏封面元数据
- 顺序播放、单曲循环、随机播放三种互斥播放模式
- 基于 Media3 的真实本地播放，以及通知栏和锁屏媒体控件
- MediaStore 与 SAF 扫描 MP3、AAC/M4A、FLAC、WAV、OGG/Opus

## 构建与安装

```bash
./gradlew :app:assembleDebug
```

生成的可安装 Debug APK 位于 `app/build/outputs/apk/debug/app-debug.apk`。首次启动时授权音频权限；如拒绝，可选择包含音乐的本地文件夹。

## 设计系统

- 24 个原型页面、20 个组件族、28 个图标原子
- `Library Light` / `Player Dark` 双颜色模式
- `Noto Sans SC` + `Outfit`
- 触控目标不小于 48dp；状态同时使用颜色、图标与文字表达
- 167 条原型反应，包括导航、覆盖层、变量更新、组件变体切换和拖拽

## 边界

- Figma 中的扫描、播放、进度、通知和锁屏均为模拟；Android APK 则实现本地扫描与真实播放。
- `.lrc` 支持同名文件自动匹配和逐行同步；无时间轴歌词会以静态文本展示。
- 在线补全默认关闭，首次使用需授权；个人原型可直连 LRCLIB、MusicBrainz 和 Cover Art Archive，服务不可用时回退本地缓存或原创占位图。
- 在线补全会在扫描后自动采用高置信度结果；低置信度歌词或封面进入 Review 候选，可逐项确认或一次性接受全部候选后覆盖本地结果。
- 在线歌词与远程封面受各服务条款约束，当前实现面向个人原型，不承诺公开或商业再分发授权。
- 不包含账号、云同步、推荐、广告、MV、社交、均衡器、倍速、睡眠定时、文件写回、M3U 或 Android Auto。
- Code Connect 尚未配置；Android/Compose 实现已位于 `app` 模块。

原创封面与品牌资产位于 [`design-assets`](./design-assets)。
